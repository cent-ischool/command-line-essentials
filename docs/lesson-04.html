Lesson 4: GUI ↔ Terminal Bridge

The terminal and your file manager (File Explorer, Finder, Files) are just two different views of the same folders. This lesson shows you how to move between them.

The Key Insight

When you create a file in the terminal, the file manager shows it. When you delete a file in the file manager, the terminal reflects that too. They are two windows into the same folder.

Open the GUI from the Terminal

Use . to mean "the current folder":

explorer . — opens the current folder in File Explorer.

open . — opens the current folder in Finder.

xdg-open . — opens the current folder in Files (Nautilus).

You can also open a specific folder:

explorer C:\Users\YourName\Documents

open ~/Documents

xdg-open ~/Documents

Open the Terminal from the GUI

Windows 11: Right-click a folder → "Open in Terminal". Windows 10: Click the address bar → type cmd → Enter.

System Settings → Keyboard → Keyboard Shortcuts → Services → enable "New Terminal at Folder". Then right-click any folder → Services → "New Terminal at Folder".

Right-click a folder → "Open in Terminal". Or press Ctrl+Alt+T to open a terminal at your home folder.

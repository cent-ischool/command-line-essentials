# Command Line Essentials — Course Design Spec

## Overview

A 1-2 hour self-paced online course teaching absolute beginners how to use the command line, navigate files and folders, and understand the GUI↔terminal connection. Runs on Windows, macOS, and Ubuntu Linux.

**Target students:** Beginners with no prior command-line experience.
**Prerequisites:** Ability to open a terminal application on their OS.

---

## Course Structure

### Format
- **Delivery:** GitHub Pages web portal (static site built from markdown)
- **Access:** Linked from Blackboard; students navigate to GitHub Pages
- **Sequence:** Fully linear — must complete each lesson before advancing
- **Pacing:** Self-paced; no time limits

### Lesson Modules (5 lessons, ~15-25 min each)

| # | Title | Core Skills |
|---|-------|-------------|
| 1 | **What Is the Terminal?** | Open terminal on Win/Mac/Linux, understand prompt, run first command |
| 2 | **Navigating Folders** | `cd`, `ls`/`dir`, `pwd`/`cd`, relative vs absolute paths |
| 3 | **Files and Folders** | `mkdir`, `touch`, `rm`, `cp`, `mv`, naming conventions |
| 4 | **GUI ↔ Terminal Bridge** | `explorer .` / `open .` / `xdg-open .`, "Open in Terminal" from GUI |
| 5 | **Challenge: Build a Mini Project** | Apply all skills to create and organize a small project folder |

### Hint System
- Each exercise has 0-3 progressive hints
- Hints revealed via "Show Hint" button (no penalty)
- Designed to unblock without giving away the answer

### Badge / Certificate System

Three outputs on completion:

1. **GitHub Actions checkmark** — automated test suite runs challenge commands; green checkmark in repo
2. **Badge SVG image** — generated by Actions, downloadable from the repo releases
3. **Certificate PDF** — generated with student name, date, and badge image

### How It Works

1. Student forks/clones the repo
2. Opens the GitHub Pages URL (linked from Blackboard)
3. Reads lesson in browser, does exercises in their local terminal
4. Runs a verification command (e.g., `bash scripts/verify.sh 01`) to check their work
5. On completing all 5 lessons, GitHub Actions runs the full test suite
6. If all pass: Actions outputs a green checkmark, generates badge SVG, generates cert PDF
7. Student downloads badge/cert from the Actions artifacts or repo releases

### Verification Strategy

Since students run commands on their own machines (not in a sandbox), verification is **honor-system with smoke tests**:

- Verification script checks that expected files/folders were created in the student's repo
- Challenge commands that would modify files are documented, not auto-run
- The badge check confirms the student pushed evidence (created files, commit history) rather than testing live machine state

**Rationale:** True command-line mastery is demonstrated by creating working artifacts, not by passing automated UI tests. The badge certifies that the student completed the exercises and pushed the results.

## Multi-Platform Support

All commands documented for all three OSes side-by-side:

| Concept | Windows | macOS | Ubuntu |
|---------|---------|-------|--------|
| Open terminal | Win+R → `cmd` or PowerShell | Cmd+Space → Terminal | Ctrl+Alt+T |
| List files | `dir` | `ls` | `ls` |
| Change folder | `cd` | `cd` | `cd` |
| Current directory | `cd` (no args) | `pwd` | `pwd` |
| Make folder | `mkdir` | `mkdir` | `mkdir` |
| Open GUI here | `explorer .` | `open .` | `xdg-open .` |
| Open terminal from GUI | Right-click → "Open in Terminal" | Services → "New Terminal at Folder" | Right-click → "Open in Terminal" |

## Design Principles

1. **Zero assumed knowledge** — don't assume any CS vocabulary
2. **Immediate payoff** — first lesson has students running a command within 5 minutes
3. **GUI connection first** — lesson 4 explicitly bridges the mental model gap
4. **Progressive hints** — hints guide without spoiling
5. **Platform parity** — same lesson content, platform-specific command callouts
6. **Artifacts as proof** — badge and cert come from real git history, not a quiz score
